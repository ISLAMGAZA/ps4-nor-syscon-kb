# Contributing · المساهمة

شكراً لرغبتك في المساهمة. هذا المشروع يعيش من تقارير الفنيين الحقيقية.

## Contributions welcome · أنواع المساهمة

1. **تقارير الدمبات (corpus reports)** — الأهم. أرسل تقريراً عن دمبة قمت بإصلاحها لبناء قاعدة معرفة قابلة للبحث.
2. **تحسينات الأداة** — إصلاحات أخطاء، ميزات جديدة (عبر Pull Request).
3. **الوثائق** — ترجمة، توضيح، أمثلة.

## How to submit a dump report · كيف ترسل تقرير دمبة

### الطريقة الأسهل (داخل التطبيق)
- شغّل الأداة → اختر **NOR** أو **Syscon** → بعد الفحص/الإصلاح اضغط **«Submit to corpus»**.
- يُرسل التطبيق تقريراً **مُجرّداً لا يكشف هوية الجهاز** تلقائياً (بلا serial/MAC/بيانات شخصية).

### الطريقة اليدوية (عبر Pull Request)
- أنشئ ملفاً بصيغة JSON في `community/corpus/` متبعاً شكل `community/SCHEMA.md`.
- **لا تُضمّن أبداً:** السيريال، عنوان MAC، أي معرّف جهاز، أو أي بيانات شخصية.
- افتح PR؛ سير عمل `aggregate.yml` سيحدّث إحصاءات القاعدة تلقائياً.

## Rules · القواعد
- التقارير يجب أن تكون **لا تكشف هوية الجهاز** (anonymized). أي تقرير فيه بيانات تعريف يُرفض.
- الكود مرخّص MIT؛ الوثائق CC-BY-4.0.
- كن لطيفاً وبنّاءً في النقاشات.

## Need help? · تحتاج مساعدة
راجع `SUPPORT.md` أو افتح Discussion في تبويب Discussions بالمستودع.
